### 1.2 各维度详细评估

#### 产品愿景与定位（9/10）

QuitMate 的三阶段模型（认知觉醒 → 行动改变 → 维持防复发）直接对齐了跨理论模型（TTM），这是整个戒烟戒酒领域中最被忽视的环节——**前意向阶段（Precontemplation）的意识唤醒**。市面上几乎所有产品都假设用户"已经想戒了"，而 QuitMate 从"Reality Check"（现实检验）开始，通过数据可视化让用户自己看到吸烟/饮酒的真实代价。

**产品哲学亮点**：
- Welcome Screen 提供"我正在难受"的即时 SOS 入口，不强制走完 onboarding
- "不管你是自己来的，还是被拉来的" — 共情文案打破防御心理
- Reality Check 采用"没有对错，只是事实"的非评判性策略 — 暗合动机访谈法（MI）

**战略决策**：专注 Android 平台，放弃 iOS。在中国市场这是一个务实的选择——Android 占据约 72% 的中国智能手机市场份额，且国产 ROM（华为/小米/OPPO/vivo/荣耀）兼容性已具备基础，可以集中精力把 Android 体验做到极致。

#### 架构设计（8/10）

Clean Architecture 分层（`presentation → domain → data → core`），经过 v2.0 组件拆分后架构清晰度大幅提升。

**设计亮点**：
- `HealthDataService` 抽象接口 + `SelfReportHealthService` 实现分离，为华为健康平台集成预留扩展点
- `WearableService`、`EncryptionService` 均采用抽象接口，实现可替换
- AI Coach、Pattern Analyzer、Craving Predictor、Relapse Risk Engine 各自独立

**扣分点**：
- `urge_toolkit_screen.dart` 仍有 ~1250 行，管理多个 AnimationController 和 Timer
- Repository 层直接操作 SQL，缺少 ORM 抽象
- `GameProfile` 的 `level`/`xp` 是可变字段，打破不可变性模式

### 1.3 核心优势

1. **三阶段全链路**：从"不想戒"到"戒了不复发"的完整干预路径，市面独一无二

**次要用户**：家人朋友（帮助者）、企业 HR（EAP）、戒烟门诊医生（FHIR）。

## 七、产品设计红线

1. **禁止恐吓式科普**：多做"正向收益"展示，避免引发心理防御
2. **禁止在用户脆弱时展示广告**：会摧毁信任并可能引发报复性复吸
3. **禁止强制社交绑定**：戒烟戒酒极度私密，强制社交绑定将导致 80% 流失
4. **禁止功能蔓延**：先死磕烟/酒，把三阶段模型跑通做深
5. **禁止降低隐私承诺**：网络功能必须是 opt-in + 明确隐私提示
6. **禁止"全或无"的复发处理**：破戒一次就清零 = 惩罚式设计 = 用户彻底卸载

**现状**：无状态，每次响应独立计算，无对话记忆。

**升级方案**：
- 引入 `ConversationContext`，存储最近 N 轮对话主题、情绪趋势、已讨论策略
- `generateResponse()` 读取上下文，生成连贯跟进问题
- 对话主题切换检测（渴望→睡眠 突然切换）
- 中文模糊匹配（拼音首字母、近义词）
- `ConversationContext` 持久化到 SQLite（新表 `conversation_history`）

#### 8.2.2 Urge Toolkit 状态机重构

**现状**：~1250 行，手动管理多个 Timer/AnimationController。

**重构方案**：
