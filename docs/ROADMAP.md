## 二、核心优势分析

### 2.1 产品哲学亮点

QuitMate 的三阶段干预模型（认知觉醒 → 行动改变 → 维持防复发）直接对齐跨理论模型（TTM, Transtheoretical Model），这是戒烟戒酒领域中最被忽视的环节——**前意向阶段（Precontemplation）的意识唤醒**。

市面上几乎所有竞品都假设用户"已经想戒了"，而 QuitMate 从"Reality Check"（现实检验）开始，通过数据可视化让用户自己看到吸烟/饮酒的真实代价。

**具体亮点**：
- Welcome Screen 提供"我正在难受"的即时 SOS 入口，不强制走完 onboarding
- Reality Check 采用"没有对错，只是事实"的非评判性策略 — 暗合动机访谈法（MI）
- 复发宽容机制："Slide, not a fall"，7 天滚动窗口 — 国内同类产品中独一无二

### 2.2 技术优势

| 优势 | 说明 |
|------|------|
| 纯离线架构 | AES-256-CBC 加密 + 零 Google 依赖，完美适配国产 ROM |
| 端侧 AI 引擎 | 纯 Dart 时间序列预测（渴望预测 + 复发风险评估），零外部依赖 |
| Clean Architecture | Domain 层零框架依赖，Data/Presentation 各司其职 |
| 抽象接口设计 | HealthDataService/WearableService/EncryptionService 均可替换实现 |
| 专业设计系统 | ThemeExtension 语义色彩，零硬编码颜色，31KB 主题配置 |
| 国产 ROM 适配 | 华为/小米/OPPO/荣耀通知权限、存储权限、PIPL 合规 |

### 8.2 P1：AI 教练深度升级（第 3-6 周）

#### 8.2.1 AI Coach 有状态对话

**现状分析**：
- 已有 `conversation_context.dart` 和 `ai_coach_engine_v2.dart`，框架已搭好

## 十、产品设计红线

以下规则在任何版本迭代中均不可违反：

| # | 红线 | 原因 |
|---|------|------|
| 1 | **禁止恐吓式科普** | 多做"正向收益"展示，避免引发心理防御导致用户流失 |
| 2 | **禁止在用户脆弱时展示广告** | 用户在渴望发作时最脆弱，此时展示广告将摧毁信任 |
| 3 | **禁止强制社交绑定** | 戒烟戒酒极度私密，强制社交绑定将导致 80%+ 流失 |
| 4 | **禁止功能蔓延** | 先死磕烟/酒，把三阶段模型跑通做深，再考虑扩展 |
| 5 | **禁止降低隐私承诺** | 网络功能必须是 opt-in + 明确隐私提示，不可静默上传数据 |
| 6 | **禁止"全或无"的复发处理** | 破戒一次就清零 = 惩罚式设计 = 用户彻底卸载 |
| 7 | **禁止依赖 Google 服务** | 目标用户群使用国产 ROM，GMS 不可用将导致功能失效 |
