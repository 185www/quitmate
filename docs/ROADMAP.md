### 1.2 各维度详细评估

#### 产品愿景与定位（9/10）

QuitMate 的三阶段模型（认知觉醒 → 行动改变 → 维持防复发）直接对齐了跨理论模型（TTM），这是整个戒烟戒酒领域中最被忽视的环节——**前意向阶段（Precontemplation）的意识唤醒**。市面上几乎所有产品都假设用户"已经想戒了"，而 QuitMate 从"Reality Check"（现实检验）开始，通过数据可视化让用户自己看到吸烟/饮酒的真实代价。

**产品哲学亮点**：
- Welcome Screen 提供"我正在难受"的即时 SOS 入口，不强制走完 onboarding — 体现了"用户需求优先"的设计理念
- "不管你是自己来的，还是被拉来的" — 共情文案打破防御心理
- Reality Check 采用"没有对错，只是事实"的非评判性策略 — 暗合动机访谈法（MI）的核心原则

**扣分点**：Reality Check 仍然偏重数字输入（每日消耗量/单价），缺少视觉冲击力更强的沉浸式体验（如 AR 肺部对比、第一人称身体变化动画等前沿交互）。

#### 架构设计（8/10）

采用了 Clean Architecture 分层（`presentation → domain → data → core`），这在 Flutter 社区中属于高质量实践。经过 v2.0 的组件拆分，架构清晰度大幅提升。

**设计亮点**：
- **领域驱动**：`UserStage` 枚举直接映射 TTM 模型的 5 个阶段
- **策略模式**：`HealthDataService` 抽象接口 + `SelfReportHealthService` 实现分离，为华为/Apple 健康平台集成预留了干净扩展点
- **接口先行**：`WearableService`、`EncryptionService`、`HealthDataService` 均采用抽象接口，实现可替换
- **关注点分离**：AI Coach、Pattern Analyzer、Craving Predictor、Relapse Risk Engine 各自独立，可独立测试

**扣分点**：
- `urge_toolkit_screen.dart` 仍有 ~1250 行，内部管理多个 `AnimationController` 和 `Timer`，建议引入状态机模式
- Repository 层直接操作 SQL，缺少 Drift/ORM 等数据访问抽象层
- `GameProfile` 的 `level` 和 `xp` 是可变字段，打破了不可变性模式

#### 功能完整度（8/10）

功能链覆盖完整，从意识唤醒到复发预防的全链路已打通：

**最突出的特色**：
- **复发宽容机制**（Slide, not a fall）：7 天滚动窗口，longestStreak 永远保留 — 这在国内同类产品中独一无二
- **渴望预测引擎**：纯 Dart 实现的时间序列预测，指数平滑 + 逐小时分布 + 置信度 sigmoid 评分
- **复发风险评估**：7 因子加权模型，渴望因子权重最高（0.25）— 符合临床观察

#### 代码质量（7/10）— 较 v5.0 提升 +1

**亮点**：
- **设计系统极其专业**（`app_theme.dart` 31KB）：`ThemeExtension` 实现语义色彩系统，零硬编码颜色，卡片 elevation: 0 + 0.5px 边框的现代扁平风格，灵感受到 Headspace/Calm/Noom/Fabulous 启发
- **组件拆分成果显著**：Dashboard 从 45KB→292 行（-76%），Skills Lab 从 51KB→196 行（-83%）
- **ML 引擎代码质量优秀**：`CravingPredictor` 和 `RelapseRiskEngine` 纯 Dart、零依赖、完全可测试，文档完整
- **加密实现规范**：AES-256-CBC + 随机 IV + Keystore/SQLite 双存储策略（兼容 HarmonyOS NEXT）

- AI Coach 是无状态的（无对话历史记忆），中文关键词匹配没有容错

- CONTRIBUTING.md 提及 BLoC 模式，但实际代码使用 Riverpod — 文档不一致
- SQLite 未启用 `PRAGMA foreign_keys = ON`，缺少事务支持

### 1.3 核心优势

1. **三阶段模型全链路覆盖**：从"不想戒"到"戒了不复发"的完整干预路径，在市面上独一无二
2. **纯离线 + 隐私优先**：AES-256 加密 + PIPL 合规 + 零 Google 依赖（适配国产 ROM），差异化卖点极强
3. **科学方法论扎实**：TTM 模型 + FTND/AUDIT 量表 + CBT 技能库 + MI 动机访谈 + WHO/CDC 健康里程碑
4. **游戏化体系精良**：指数 XP 曲线 + 10 级等级体系 + 虚拟植物 + 储蓄罐动画 — 心理学驱动的激励设计
5. **端侧 AI 引擎**：渴望预测（指数平滑 + sigmoid 置信度）和复发风险评估（7 因子加权）— 纯 Dart 实现，零外部依赖
6. **复发宽容哲学**："Slide, not a fall"机制在国内戒烟应用中首创，避免了"惩罚式设计"导致用户流失
7. **设计系统专业**：Headspace/Calm 级别的 Nature-infused 设计语言 + 完整的 ThemeExtension 语义系统

### 目标用户画像

**核心用户**：25-45 岁的中国男性，有吸烟或饮酒习惯，但处于"知道不好但还没准备好戒"的犹豫状态。他们可能曾经尝试过戒烟/戒酒但失败了，对"被教育"感到厌倦。

**次要用户**：
- 家人朋友希望帮助亲人戒烟戒酒的"关心者"群体
- 企业 HR（通过 EAP 模式批量采购）
- 戒烟门诊医生（通过 FHIR 导出获取患者数据）

## 七、产品设计红线

> 基于对市场竞品的深度分析和行为心理学原理，以下设计决策必须坚决避免：

1. **禁止恐吓式科普**：过度展示烂肺、黑肝图片会引起心理防御机制，导致用户逃避。应多做"正向收益"展示。
2. **禁止在用户脆弱时展示广告**：烟瘾/酒瘾发作时的焦虑状态下推送广告，会瞬间摧毁信任并可能引发报复性复吸。
3. **禁止强制社交绑定**：戒烟戒酒是极度私密的事情，强制绑定微信/朋友圈将导致 80% 用户在注册第一步流失。
4. **禁止功能蔓延**：不要同时做戒烟、戒酒、戒色、减肥、戒赌。先死磕烟/酒，把三阶段模型跑通做深。
5. **禁止降低隐私承诺**：任何涉及网络请求的功能必须是用户"主动选择开启"的 opt-in 模式，且必须在 UI 上有明确的隐私提示。
6. **禁止"全或无"的复发处理**：用户破戒一次就清零所有数据、让用户从零开始，这种"惩罚式设计"会导致用户自暴自弃彻底卸载 App。

## 八、v6.0 详细规划

### 8.1 P0：基础设施修复（第 1-2 周）

#### 8.1.1 CI 流水线修复

**现状**：`ci.yml` 中 `branches: ain, develop]` 缺少 `[`，CI 可能从未在 push 时触发。

**修复方案**：
```yaml
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
```

**额外优化**：
- 简化 coverage regex（用 `lcov` 替代 400+ 字符 grep）
- 添加 `flutter analyze --fatal-infos` 严格模式
- 添加 APK size diff 对比（每次 CI 记录包体积变化）

#### 8.1.2 iOS 实际构建

### 8.2 P1：体验升级（第 3-6 周）

#### 8.2.1 AI Coach 有状态对话

**现状**：`AiCoachEngine` 是无状态的，每次响应都是独立计算，缺少对话记忆和个性化上下文。

**升级方案**：
- 引入 `ConversationContext` 数据类，存储最近 N 轮对话的主题、情绪趋势、已讨论的策略
- 在 `generateResponse()` 中读取对话上下文，生成连贯的跟进问题
- 实现对话主题切换检测（用户从讨论渴望突然转而讨论睡眠）
- 添加模糊匹配（拼音首字母、近义词）提升中文关键词识别率
- `ConversationContext` 持久化到 SQLite（新表 `conversation_history`）

#### 8.2.2 Urge Toolkit 状态机重构

**现状**：`urge_toolkit_screen.dart` ~1250 行，手动管理多个 `Timer` 和 `AnimationController`。

**重构方案**：
- 引入 `UrgeToolkitState` 枚举（idle / delay / breathing / surfing / grounding / completed）
- 使用 `StateMachine` 模式管理状态转换
- 将每个工具（延迟计时/呼吸法/冲浪/接地法）拆分为独立的 `StatefulWidget`
